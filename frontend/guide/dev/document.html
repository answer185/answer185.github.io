# 文档规范

# 概述

文档是前端团队除代码外的另一个重要沉淀。

因为项目越来越庞大，模块已经达到6000左右，如果没有文档支撑，维护和开发难度很大。

文档的工具在经历了2019年gitbook和docsify后，最终确定使用vuepress

书写的内容优先代码管理、公共组件等

# 书写

要求团队成员学会markdown语法

规范参考 antd

书写的时候，请执行

yarn start

启动本地服务

# 部署

线上的文档域名是http://frontend-docs.hecom.cn/, 访问的时候，需要账户密码

通过walle上传，对应的仓库是web / frontend-docs-dist

文档的仓库是 web / frontend-docs-source

拆分成两个，主要是考虑markdown项目之后图片，md等文件会较多，避免上传两次。

两个仓库请放在同一个文件夹，执行

yarn run build

后会自动把生成的文件放到 web / frontend-docs-dist 中
