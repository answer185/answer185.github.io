# 代码规范

# 概述

为了减少bug 和 降低维护成本，特引入eslint、stylelint及prettier这3个业内主流的工具。

prettier通过插件的方式，作为一个规则与两个工具整合。

# 基础规则

eslint的基础规则是airbnb、prettier 及相应的推荐设置

stylelint的基础规则是stylelint-config-standard、 stylelint-config-rational-order 和 stylelint-config-prettier

# 自定义规则

目前只设置了缩进是2个空格，之后根据团队的实际情况扩展

# 开发工具

目前主要的开发工具是WebStorm和VSCode。

因为在项目初期没来得及引入代码规范, 所以现在代码风格很多。为避免冲突，开发工具不能开启自动格式化的功能，避免不必要的代码冲突。

预期的效果是编辑器中可以显示出不合规范的位置，并通过单独设置的快捷方式纠正，即要求手动触发

# WebStorm 的配置

WebStorm 本身已经内置支持eslint和stylelint，且默认不开启自动格式化。

eslint的快捷键也是内置，alt+L

stylelint需要通过Extend Tools配置， 然后在keymap中搜索到该工具，设置快捷方式, 建议 ctrl+shift+l

配置参数如下：

Program: $ProjectFileDir$\node_modules\stylelint\bin\stylelint.js
Arguments: $FileName$ --fix
Working Director: $FileDir$

# 包

因为历史原因，项目目前最高只支持node 8.9.1，所以对一些包的版本做了降级。 具体版本见package.json

名称 作用
babel-eslint eslint的babel解析器
eslint eslint
eslint-config-airbnb airbnb的规范
eslint-config-prettier 使用prettier的规则覆盖eslint，解决冲突问题
eslint-plugin-babel 使babel-eslint支持es6的实验性特性
eslint-plugin-import 使eslint支持es6的import export语法
eslint-plugin-jest jest的规则
eslint-plugin-jsx-a11y JSX元素的检测规则
eslint-plugin-react react的规则
eslint-plugin-react-hooks hooks的规则（暂时没用）
eslint-plugin-prettier 把prettier作为eslint的规则运行，使按照eslint的规则来报
prettier prettier
stylelint stylelint
stylelint-config-prettier 把可能与prettier冲突的规则关闭掉
stylelint-config-standard stylelint-config-recommended的扩展配置
stylelint-declaration-block-no-ignored-properties 避免设置冲突的css属性 display: inline; width: 100px;, width在inline下，会被 忽略
stylelint-order css排序
stylelint-config-rational-order css的排序
stylelint-config-css-modules css模块化的写法配置

# 配置文件

名称 作用
.editorConfig 编辑器的配置文件
.eslintignore eslint忽略的文件
.eslintrc eslint配置
.prettierignore prettier不处理的文件
.prettierrc prettier的配置
.stylelintrc stylelint的配置文件
